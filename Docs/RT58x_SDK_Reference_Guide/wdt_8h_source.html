74 /*Remark:
75  * WDT interrupt is "warning interrupt", that notify system it will reset soon.
76  * so int_ticks should be less that wdt_ticks and should have enough time for
77  * "reset error" handling.. int_ticks can NOT greater or equal then wdt_ticks.
78  *
79  * If you set lock_enable mode, you can NOT change the watchdog setting anymore
80  * before reset (after WDT lock_enable bit has been set). That is you must kick
81  * the watchdog periodic to prevent reset event.
82  *
83  * Example:
84  * Clock is 32M Hz. Set prescale is 32.. so WDT is 1M Hz.
85  * If we set wdt_ticks 500000, and int_ticks 300
86  * then we should kick WDT within 0.5 second, otherwise
87  * the whole system will reset (if reset enable) after 0.5s
88  * from the last kick and before reset. And before reset,
89  * a warning reset interrupt will generate, system has 300 us
90  * to save some important data or something..
91  * We can ignore int_ticks and wdt_handler, if we don't set
92  * int_enable mode.
93  *
94  */
