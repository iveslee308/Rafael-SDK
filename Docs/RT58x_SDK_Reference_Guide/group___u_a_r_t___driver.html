◆ uart_rx()

uint32_t uart_rx ( uint32_t  uart_id,
uint8_t *  p_data,
uint32_t  length 
)

Function for receiving data over UART.

If an event handler was provided in the uart_init() call, this function returns immediately and the handler is called when the transfer is done. Otherwise, the transfer is performed in blocking mode, i.e. this function returns when the transfer is finished.

During the operation it is not allowed to call this function or any other data receive function again in non-blocking mode. Also the data buffers must stay allocated and the contents of data must not be modified.

Status of the receiver can be monitored by calling the function "uart_status_get" and checking tge rx_busy flag which indicates if reception is still in progress.

Parameters
[in]uart_ididentifier for uart device.
[in]p_dataPointer to data.
[in]lengthNumber of bytes to receive. Maximum possible length is 4095
Return values
STATUS_SUCCESSReceive request was successful. For non-block mode, data is in receiving state.
STATUS_INVALID_PARAMWrong parameter, like uart_id is invalid number length size is too large or zero.
STATUS_NO_INITuart device did NOT configure yet.
STATUS_EBUSYuart device is in previous receiving state.
