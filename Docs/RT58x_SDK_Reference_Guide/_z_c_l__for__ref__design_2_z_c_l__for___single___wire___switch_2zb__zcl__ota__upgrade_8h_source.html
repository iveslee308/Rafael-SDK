149 /*
150  OTA Upgrade Spec (6.3.2.7 File Version) has dome recommendations about versioning (at least to
151  divide version to 4 parts - Stack Build, Stack Release, Application Build and Application
152  Release), but does not specify comparing method. Looks like we do not have any limitations and
153  may upgrade/downgrade any f/w (except re-installation), so this comparing is simply:
154  ver1 != ver2
155  Only one possible limitation now is NVRAM, but if dataset ids are not overlapped, it is also ok.
156  Any additional comparison logic may be implemented on the application level.
157  */
159 #define ZB_ZCL_OTA_UPGRADE_VERSION_CMP(ver1, ver2) \
160  ( (ver1) != (ver2) )
